1. 產品概述

1.1 產品背景

本平台為成人娛樂聚合站（以下簡稱「色站」），以成人影片內容為核心流量入口，整合直播聊天室、遊戲玩家自製影片（UGC）與遊戲廣告跳轉功能，實現成人內容站與遊戲運營商平台的雙向導流。 平台同時支援 C 端訪客 / 會員的內容消費行為，以及 B 端運營人員的內容稽核、廣告配置與數據分析。

1.2 產品定位

核心定位：以成人影片（免費 + UGC）＋直播為留存手段，以遊戲廣告跳轉為主要商業變現路徑，形成「內容消費 → 廣告曝光 → 遊戲站引流 → 遊戲站反哺導流」的流量閉環。

1.3 目標用戶

角色描述核心訴求
C端 訪客 未注冊用戶，以搜索引擎或廣告連結進入 快速瀏覽免費影片，無摩擦進入體驗
C端 一般會員 已注冊，可收藏、留言、觀看更多內容 個人化內容推薦、社群互動（聊天室）
C端 VIP 會員 付費或遊戲站導入的高價值用戶 高清無廣告影片、直播互動、專屬福利
C端 遊戲玩家 從遊戲站跳轉進入、或本站上傳影片的玩家 上傳 / 分享影片、獲得站內曝光回饋
B端 運營人員 管理內容、廣告、用戶的後台操作人員 高效稽核影片、彈性配置廣告位、查看報表

1.4 範圍說明

本版本（v1.0）涵蓋 MVP 核心功能，不含打賞禮物系統細節、付費點數商城、推薦算法引擎的詳細設計（後續版本迭代）。

4.3 遊戲玩家影片區（UGC）

此區塊展示由遊戲玩家（來自遊戲站的用戶）上傳的影片，與免費影片區在視覺上有差異化設計（如加上「玩家上傳」標籤），以強化色站與遊戲站的用戶社群連結。

4.3.1 展示邏輯

  • 列表頁獨立 Tab 或首頁推薦區塊入口
  • 展示玩家暱稱（可顯示其遊戲站用戶名或匿名），點擊可查看該玩家其他上傳作品
  • 影片卡片標示「玩家上傳」徽章，區別於官方免費影片

4.3.2 上傳流程（C端）

  • 需登入本站帳號（可用遊戲站帳號 SSO 登入或獨立注冊）
  • 上傳限制：單檔最大 2GB，支援 MP4 / MOV 格式，時長 30 秒 ~ 60 分鐘
  • 填寫：標題（必填）、封面（可自動截幀或手動上傳）、分類標籤（最多 5 個）、簡介（選填）
  • 提交後進入審核，審核通過後自動上架；審核中影片僅上傳者可見
  • 上傳成功加入積分（可兌換平台福利，如 VIP 天數）

4.4.1 直播列表頁

4.5 遊戲廣告位與跳轉機制

核心商業邏輯：色站作為流量來源，透過廣告位與跳轉機制將高意願用戶導入遊戲站，並透過推廣碼追蹤歸因，實現分潤結算；遊戲站同樣在其站內設有色站入口，形成雙向流量閉環。

4.5.1 廣告位類型

廣告位名稱位置尺寸建議展示邏輯
首頁主 Banner首頁頂部輪播1200×300px輪播 3~5 個廣告，每 5 秒切換，可點擊跳轉
影片播放前貼片影片播放前全螢幕覆蓋5 秒強制 + 可跳過，VIP 免除
影片播放中插片影片播放中（每 15 分鐘）全螢幕覆蓋強制 3 秒，不可跳過
影片推薦列表插卡相關推薦影片清單中同影片卡片尺寸每 5 筆影片插入 1 筆廣告卡片
直播間側欄 Banner直播間右側固定300×250px常駐顯示，不影響直播觀看
直播間進場彈窗進入直播間時500×300px每次進場顯示一次，3 秒後可關閉
頁尾固定廣告列所有頁面底部全寬 × 60px懸浮固定，含遊戲站 Logo + CTA 按鈕

4.5.2 跳轉機制

  • 推廣碼帶入：用戶點擊廣告後，跳轉 URL 自動附帶色站推廣碼（e.g. ?ref=xxxx&src=adult_banner），遊戲站讀取後記錄歸因
  • 跳轉方式：預設開新分頁（target="_blank"），確保用戶在色站的瀏覽session不中斷
  • 落地頁：可由 B端配置跳轉至遊戲站的注冊頁 / 活動頁 / 首頁（不同廣告位可配置不同落地頁）
  • 遊戲站回導：遊戲站在其頁面設有色站入口 Banner / 連結，同樣攜帶遊戲站的推廣碼跳轉回色站

4.5.3 廣告追蹤埋點

  • 廣告曝光次數（Impression）：每次廣告進入可視區域計為一次曝光
  • 廣告點擊次數（Click）：記錄來源廣告位 ID + 用戶 ID（或匿名 ID）
  • 跳轉至遊戲站次數（Redirect Out）：附帶推廣碼的跳出量
  • 由遊戲站回導進入次數（Redirect In）：遊戲站帶推廣碼進入色站的量

4.6 用戶系統（注冊 / 登入 / VIP）

4.6.1 注冊 / 登入

  • 支援 Email + 密碼注冊（無需實名，保護用戶隱私）
  • 支援遊戲站帳號 SSO 跳轉登入（可選串接）
  • 登入後保留 7 天 Session，可手動登出
  • 忘記密碼透過 Email 驗證重設

4.6.2 個人中心

  • 我的收藏（影片 / 直播間）
  • 我的上傳（玩家影片列表 + 審核狀態）
  • 積分與 VIP 狀態
  • 觀看記錄

4.6.3 VIP 機制

等級獲取方式權益
訪客無需注冊觀看免費影片（最高 720P）、看廣告、看直播（不可發言）
一般會員免費注冊收藏 / 留言 / 聊天室發言 / 上傳玩家影片
VIP 會員付費升級 or 從遊戲站入金後由遊戲站發放 VIP 碼兌換1080P 高清、無廣告、私密直播間、專屬內容解鎖
關鍵設計：遊戲站可透過「用戶在遊戲站完成首次入金 → 發放色站 VIP 兌換碼」的機制，激勵色站用戶在遊戲站注冊並入金，形成更強的轉化閉環。

5. B端運營後台功能需求

5.1 內容管理（影片稽核與即時控制）

5.1.1 影片列表管理

  • 即時影片狀態控制：支援在 B端一鍵「上架/下架」影片，C端前台即時同步隱藏/顯示該影片
  • 篩選：全部 / 待審核 / 已上架 / 已下架 / 審核拒絕
  • 支援批量操作：批量上架 / 下架 / 刪除
  • 搜尋：按標題 / 上傳者 / 標籤搜尋

5.1.2 影片審核流程

用戶上傳影片 ──▶ 系統自動轉碼 ──▶ 進入審核佇列 │ ▼ B端審核員查看影片 + 封面 + 標籤 │ ├──▶【通過】──▶ 上架，通知用戶，加入積分 │ └──▶【拒絕】──▶ 選擇拒絕原因 ──▶ 通知用戶 │ └── 拒絕原因：含未成年疑慮 / 非成人內容 / 侵權 / 其他

5.1.3 官方影片管理

  • 後台直接上傳官方免費影片（批量上傳）
  • 可設定影片上架時間（定時上架）
  • 管理標籤分類體系（新增 / 停用 / 合併標籤）
  • 設定影片是否含廣告插片

5.2 直播管理

  • 主播帳號管理：新增 / 停用主播帳號，設定串流密鑰（Stream Key），查看主播歷史直播記錄
  • 直播間即時控制：強制開播/下播（C端即時同步顯示直播狀態）、查看當前在線人數、全局禁言特定用戶
  • 廣告位配置：設定直播間內顯示哪個廣告（可覆蓋全站預設廣告）
  • 敏感詞庫維護：新增 / 刪除聊天室敏感詞，觸發後自動攔截

5.3 廣告與跳轉管理

5.3.1 廣告位配置

配置項說明
廣告位選擇選擇欲配置的廣告位（首頁 Banner / 貼片 / 插卡 / 直播間等）
廣告素材上傳上傳圖片 / GIF / 短視頻廣告素材
跳轉 URL 設定填寫跳轉目標 URL（遊戲站落地頁），推廣碼自動附帶或手動填入
展示排期設定廣告開始 / 結束時間，到期自動停止展示
輪播權重多個廣告輪播時，可設定各廣告的曝光比例

5.3.2 跳轉連結管理

  • 建立各遊戲站的跳轉連結（含推廣碼），統一管理多個遊戲站合作方
  • 支援 UTM 參數自動拼接（包含活動名稱、推廣渠道等參數即時產生器）
  • 可設定某廣告位的跳轉目標（從下拉選單選擇已建立的跳轉連結），並在 B端進行即時預覽與測試

5.4 用戶管理

  • 用戶列表：搜尋用戶（Email / 暱稱）、查看注冊時間、VIP 狀態、上傳數量、最後登入
  • VIP 手動發放與全局控制：輸入 Email 或用戶 ID 直接升級 VIP，或一鍵切換全局 VIP 測試狀態
  • 封禁 / 解封：封禁違規用戶（C端即時攔截互動），封禁後用戶無法登入；支援永久封禁或指定天數
  • 聊天室禁言：針對特定用戶設定聊天室禁言（C端即時生效），不影響其他功能使用

6. 非功能需求

6.1 性能

  • 首頁 LCP（最大內容渲染）≤ 2.5 秒（CDN 加速）
  • 影片秒開率 ≥ 90%（首幀加載 ≤ 1 秒）
  • 直播延遲 ≤ 3 秒（採用 HLS 或 RTMP 推流 + 低延遲 CDN）
  • 聊天室訊息延遲 ≤ 1 秒（WebSocket 長連接）
  • B端後台報表查詢響應 ≤ 5 秒

6.2 安全

  • 用戶密碼 bcrypt 加密儲存
  • 影片 CDN URL 簽名防盜鏈（Token 有效期 1 小時）
  • 上傳影片進行病毒掃描
  • 廣告跳轉 URL 防 Open Redirect 驗證（僅允許已授權域名）
  • B端後台角色權限分離（超管 / 審核員 / 廣告管理員 / 只讀報表）

6.3 合規

注意：平台應根據目標市場的法規進行合規設計，包含但不限於：年齡驗證機制、違規內容舉報處理流程、隱私政策聲明（Cookie 使用告知）、DMCA 侵權處理程序。具體合規細節依目標市場法規由法務確認後補充。
  • 進站年齡確認彈窗（需用戶主動確認 18 歲以上）
  • 頁尾顯示合規聲明文字
  • 提供影片下架申請入口（DMCA / 侵權舉報）
  • 隱私政策頁面說明 Cookie 收集範圍

6.4 設備支援

  • 響應式設計（RWD），支援 PC / Mobile H5 / Tablet
  • 瀏覽器支援：Chrome 90+、Safari 14+、Firefox 88+、Edge 90+
  • 不開發原生 App（v1.0），Mobile 以 H5 為主
